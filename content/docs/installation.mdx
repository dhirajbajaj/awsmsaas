<Steps>

### Create project

Start by unzipping awsmsaas code in your local;

Configure/Setup your project to run on localhost & then deploy to vercel as it will require configurations 

<Callout type="warning" twClass="mt-4">
  Once configured completely, You can deploy to Vercel from your github repo directly :
  [![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fusername%2Fawsmsaas)
</Callout>

### Install dependencies

Enter in the folder and install dependencies for your project:

```bash
cd awsmsaas
pnpm install
```

Install pnpm if not installed; its faster node package manager than npm and uses less memory
```bash
npm install -g pnpm
```

### Create a `.env` file

Copy/paste the `.env.example` in the `.env` file:

```bash
cp .env.example .env
```


```md title=".env"
NEXT_PUBLIC_APP_URL=http://localhost:3000

AUTH_SECRET=
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=

DATABASE_URL=

RESEND_API_KEY=
EMAIL_FROM=

STRIPE_API_KEY=
STRIPE_WEBHOOK_SECRET=

NEXT_PUBLIC_STRIPE_PRO_MONTHLY_PLAN_ID=
NEXT_PUBLIC_STRIPE_PRO_YEARLY_PLAN_ID=
NEXT_PUBLIC_STRIPE_BUSINESS_MONTHLY_PLAN_ID=
NEXT_PUBLIC_STRIPE_BUSINESS_YEARLY_PLAN_ID=
```

### Configuration part

Let's check the configuration part for update all environment variables before use `pnpm run dev`.
